# 탐지 룰: SSH Brute Force

## 요약

동일한 출발지 IP에서 반복적으로 발생하는 SSH 인증 실패를 탐지합니다.

이 룰은 인터넷에 노출된 Linux 서버에서 SSH 무차별 대입 공격 가능성을 식별하기 위한 탐지 룰입니다.

## 위협 설명

SSH Brute Force는 공격자가 서버에 접근하기 위해 사용자 이름, 비밀번호 또는 둘 다를 반복적으로 추측하는 공격입니다.

SSH가 인터넷에 노출되어 있고 비밀번호 기반 로그인이 허용되어 있다면, 반복적인 로그인 실패는 서버 계정 탈취 시도를 의미할 수 있습니다.

## 로그 소스

| 항목 | 값 |
|---|---|
| 로그 소스 | Linux 인증 로그 |
| 예시 경로 | `/var/log/auth.log` |
| 이벤트 유형 | SSH 로그인 실패 |
| 서비스 | `sshd` |

## 탐지 로직

동일한 출발지 IP에서 SSH 로그인 실패가 5회 이상 발생하면 보안 이벤트로 탐지합니다.

```text
source_ip 기준 Failed SSH login count >= 5
```

## 로그 패턴

예시 로그:

```text
May 19 10:01:01 ip-10-10-1-10 sshd[1001]: Failed password for invalid user admin from 203.0.113.10 port 53211 ssh2
```

탐지 스크립트는 다음 부분에서 출발지 IP를 추출합니다.

```text
from 203.0.113.10 port
```

## 탐지 결과 예시

```json
[
  {
    "type": "SSH_BRUTE_FORCE",
    "source_ip": "203.0.113.10",
    "failed_count": 5,
    "severity": "HIGH",
    "recommendation": "Investigate the source IP and restrict SSH access"
  }
]
```

## 심각도 기준

| 심각도 | 조건 |
|---|---|
| Low | 실패 1회 이상 2회 이하 |
| Medium | 실패 3회 이상 4회 이하 |
| High | 동일 IP에서 실패 5회 이상 |

현재 MVP에서는 High 조건에 해당하는 이벤트만 알림으로 출력합니다.

## 대응 절차

탐지 발생 시 다음 절차를 따릅니다.

1. 출발지 IP를 확인합니다.
2. 시도된 사용자 이름이 실제 시스템 계정인지 확인합니다.
3. 실패 이후 성공한 SSH 로그인이 있었는지 확인합니다.
4. SSH 접속 허용 IP를 신뢰할 수 있는 IP로 제한합니다.
5. 가능하면 비밀번호 기반 SSH 로그인을 비활성화합니다.
6. 키 기반 인증을 사용합니다.
7. 악성 활동으로 판단되면 해당 IP 차단을 검토합니다.

## 오탐 가능성

다음과 같은 경우 오탐이 발생할 수 있습니다.

- 관리자가 비밀번호를 여러 번 잘못 입력한 경우
- 자동화 스크립트나 배포 도구의 인증 정보가 오래된 경우
- 모니터링 도구가 잘못된 계정 정보로 접속을 시도한 경우
- 여러 사용자가 동일한 NAT IP를 사용하는 경우

## 한계점

현재 룰은 다음 한계를 가집니다.

- 시간 윈도우 기반 탐지를 적용하지 않았습니다.
- 매우 느린 속도의 Brute Force 시도는 놓칠 수 있습니다.
- 여러 서버를 대상으로 분산된 Password Spraying은 탐지하지 못합니다.
- 로그인 실패 이후 성공한 로그인과의 상관분석은 아직 포함하지 않았습니다.
- 현재 MVP에서는 Linux 인증 로그만 분석합니다.

## 향후 개선 방향

향후 다음 기능을 추가할 예정입니다.

- 시간 윈도우 기반 탐지
- 신뢰 IP allowlist 기능
- 로그인 실패 이후 성공 이벤트와의 상관분석
- 사용자 이름 기준 분석
- 자동 대응 Dry Run 기능
- 실제 AWS EC2 로그 연동
