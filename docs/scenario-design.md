# 시나리오 설계 기준

## 개요
이 문서는 클라우드 오설정 탐지 시나리오를 어떤 기준으로 설계하는지 정리합니다.
이 프로젝트의 각 시나리오는 단순히 위험 설정을 나열하는 방식이 아니라, 오설정이 실제 보안 위협으로 이어지는 흐름을 설명하고 탐지 가능한 데이터로 연결하는 방식으로 구성합니다.

## 시나리오 구성 원칙
각 시나리오는 다음 구조를 따릅니다.

```
오설정
-> 공격 표면 또는 위험 조건
-> 발생 가능한 로그 또는 설정 데이터
-> 탐지 기준
-> 오탐 가능성
-> 개선 권고
```

## 시나리오 선정 기준
시나리오는 다음 기준으로 선정합니다.

    1. 클라우드 환경에서 자주 발생할 수 있는 설정이어야 합니다.
    2. 실제 보안 위협과 연결되어야 합니다.
    3. 로그 또는 설정 데이터로 탐지 가능해야 합니다.
    4. 탐지 기준을 명확히 문서화할 수 있어야 합니다.
    5. 개선 권고가 구체적이어야 합니다.

## 시나리오 문서 형식
각 시나리오 문서는 다음 항목을 포함합니다.

    * 시나리오 개요
    * 위험한 설정
    * 위협 흐름
    * 탐지 데이터
    * 탐지 기준
    * 오탐 가능성
    * 개선 권고
    * 구현 파일

## 탐지 유형
이 프로젝트에서 다루는 탐지 유형은 크게 두 가지입니다.

| 유형 | 설명 | 예시 |
|---|---|---|
| 설정 기반 탐지 | 클라우드 리소스 설정 자체를 점검 | Public S3, Public SSH, 과도한 IAM 권한 |
| 로그 기반 탐지 | 발생한 이벤트 로그를 분석 | SSH Brute Force, CloudTrail StopLogging |

## 목표
최종 목표는 각 시나리오에 대해 다음 질문에 답하는 것입니다.

    * 무엇이 위험한 설정인가?
    * 왜 위험한가?
    * 어떤 공격 또는 사고로 이어질 수 있는가?
    * 어떤 데이터로 탐지할 수 있는가?
    * 탐지 기준은 무엇인가?
    * 오탐은 어떻게 발생할 수 있는가?
    * 어떻게 개선해야 하는가?
